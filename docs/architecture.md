# Architecture

At a high level, the private LUX.AI stack is organized into four layers:

1. A structural layer that defines stable profile tendencies.
2. A dynamics layer that updates behavioral state over time under pressure, recovery, and memory.
3. An expression layer that turns state into communicative style and response intent.
4. A guarded rendering layer where heuristic or local-LLM text generation can be used under validation.

The public repository documents these layers conceptually, but does not expose the internal engine, full schemas, proprietary routing policies, or calibration assets.
